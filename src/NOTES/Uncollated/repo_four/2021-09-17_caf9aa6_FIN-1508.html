Description

Introduced minor refinements to the AccountEntry APIs for compatibility with the implementation in the Submission System.


Changes

Replaced the mandatory TrnReference parameters with an optional "UploadReference" parameter. This is due to the implementation that was chosen for the submission system, whereby an account entry upload may contain multiple account entries. Report Data store is still agnostic to the file contents and/or schema , though it is to be noted that the JSON to be uploaded is analogous to the Submission System XML schema - this is a requirement for the Submission System to be able to consume the data.


Other than this, the AccountEntryDownload population has been configured to add an entry for EVERY upload - regardless of the reference uniqueness.
